🚀 STEP-BY-STEP DBEAVER SETUP:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 📥 Download & Install DBeaver:
   • Go to: https://dbeaver.io/download/
   • Download "DBeaver Community Edition" (FREE)
   • Install and launch DBeaver

2. 🔗 Create New Connection:
   • Click "New Database Connection" (+) button
   • Select "PostgreSQL" from the list
   • Click "Next"

3. ⚙️ Configure Connection Settings:
   ┌─────────────────────────────────────────────────────────┐
   │ Connection Settings:                                    │
   │                                                         │
   │ Host:          [YOUR_VPS_IP]                           │
   │ Port:          5432                                     │
   │ Database:      newsdb                                   │
   │ Username:      newsuser                                 │
   │ Password:      YourSecurePassword123!                   │
   │                                                         │
   │ ☑ Save password locally                                 │
   │ ☑ Show all databases                                    │
   └─────────────────────────────────────────────────────────┘

4. 🧪 Test Connection:
   • Click "Test Connection" button
   • Should show: "Connected" with green checkmark
   • If prompted, download PostgreSQL driver

5. ✅ Finish Setup:
   • Click "Finish" to create connection
   • Connection appears in Database Navigator

💡 HOW TO VIEW & EDIT DATA:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 📊 View Table Data:
   • Right-click on "articles" table
   • Select "Open Data" (or press F8)
   • Data appears in spreadsheet view

2. ✏️ Edit Data:
   • Double-click any cell to edit
   • Press Enter to save changes
   • Changes save to database immediately

3. ➕ Add New Records:
   • Right-click in data grid
   • Select "Add Row"
   • Fill in the new row data
   • Press Ctrl+S to save

🚨 TROUBLESHOOTING:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❌ Connection Refused:
   → Run the PostgreSQL configuration script first
   → Check firewall: sudo ufw allow 5432

❌ Authentication Failed:
   → Verify password: YourSecurePassword123!
   → Check username: newsuser

❌ Database Not Found:
   → Make sure database name is: newsdb
   → Check if newsdb exists: sudo -u postgres psql -l

❌ Timeout:
   → Verify your VPS IP address
   → Check if PostgreSQL is listening: sudo netstat -tlnp | grep 5432
